# Snake Mission Log

## Captain's Intent
Demonstrate an end-to-end Starship mission in which a small crew organizes and delivers a tiny playable game using recognizable human software practices.

## Crew
- Captain
- Software Project Manager
- Coder One
- Coder Two

## Mission Flow
1. Define the game through a Game Design Document.
2. Define the implementation through a Software Design Document.
3. Derive requirements and split tasks.
4. Implement logic and tests.
5. Implement rendering and runtime orchestration.
6. Integrate and review.

## Version Control Discipline
- Planning docs committed first.
- Implementation files grouped by responsibility.
- Unit tests included as part of execution.
- Mission artifacts kept in the example workspace.
