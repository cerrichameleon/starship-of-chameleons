# Starship Example Workflow: Snake Mission

This folder is an example working area inside the Starship repository.

Its purpose is to let the Starship crew work like a small real software team:
- define the game
- define the software design
- derive requirements and tasks
- implement in a contained workspace
- use version control so crew members do not step on one another
- leave a reviewable paper trail

## Intended Crew
- Captain
- Software Project Manager
- Coder One
- Coder Two

## Working Rules
- Documentation first, then implementation
- Keep commits small and meaningful
- Separate source, tests, docs, and logs
- Record decisions in writing
- Prefer readable code and explicit names

## Folder Layout
- `docs/` design and planning documents
- `src/` implementation files for this mission
- `tests/` unit tests for this mission
- `logs/` mission outputs, status snapshots, and review artifacts
